What It Does

          
AWS on your machine, zero cloud spend

          

            

              
☁️ Local AWS Services

              
LocalStack emulates S3, SQS, Lambda, DynamoDB, and more on your machine. Develop and test without touching a real AWS account.

            

            

              
🐳 One Command Up

              
A pre-wired docker-compose.yml and Dockerfile — just docker-compose up --build and the entire stack is running.

            

            

              
⚙️ Auto-Init Scripts

              
On boot the container runs scripts/init.sh, which chains your child scripts in order to seed buckets, queues, and whatever else you need.

            

            

              
💰 Zero Cloud Cost

              
Everything runs in a throwaway container. Tear it down, rebuild, and start clean — no orphaned cloud resources, no surprise charges.

            

          

        

      


      
      

        

          
Get Started

          
Running in three steps

          

            

              

                
1

                

                  Install Docker
                  
Install Docker Desktop (includes Compose). On Linux, also install the docker-compose plugin.

                

              

              

                
2

                

                  Pull the LocalStack image
                  
Run docker pull localstack/localstack to grab the latest image once.

                

              

              

                
3

                

                  Spin it up
                  
Clone the repo and run docker-compose up --build. Init scripts provision your resources automatically.

                

              

            

            

              

                Terminal
                
              

              
# clone & enter
git clone \
  https://github.com/karantrehan3/docker-localstack
cd docker-localstack

# pull LocalStack image
docker pull localstack/localstack

# spin it all up
docker-compose up --build

            

          

        

      


      
      

        

          
Under the Hood

          
How the init pipeline works

          

            

              
🐳

              
Container starts

              
docker-compose up builds and launches the LocalStack container.

            

            

              
📄

              
init.sh runs

              
The entrypoint calls scripts/init.sh which orchestrates the pipeline.

            

            

              
⚡

              
Child scripts fire

              
init.sh chains child scripts in order — creating buckets, queues, tables, etc.

            

            

              
✅

              
Stack ready

              
All AWS resources are provisioned. Hit the local endpoints and start developing.
